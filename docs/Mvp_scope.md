# MVP 명세서 (MVP Spec v1.1 · 구현 동결본)

## 1. 제품 한 줄 정의 (One-liner)
> 이 앱은 대학생·자기관리형 사용자가 일상적인 공부·운동·작업 계획을 세우고 실행하는 과정에서  
> 계획과 실제 행동의 차이를 기록·피드백하여 다음 계획을 더 현실적으로 조정하도록 돕는  
> 경량 계획·피드백 기록 도구다.

---

## 2. 해결하려는 문제 (Problem Statement)
- 해야 할 일이 많아 계획을 세우지만 실제로는 자주 어긋난다
- 계획을 과하게 잡고 실패를 반복해 본인의 역량을 정확히 파악하지 못한다
- 피드백의 필요성은 느끼지만, 매번 텍스트로 기록하는 것은 부담이다
- 매일 계획을 처음부터 작성하는 행위 자체가 귀찮아 지속되지 않는다

※ 본 v1.0 MVP는 기능의 풍부함이 아니라,  
**계획–실행–피드백 사이클이 실제로 귀찮지 않게 돌아가는지**를 검증하는 데 목적이 있다.

---

## 3. 타겟 사용자 (Target User)
- 시험·과제·운동·개인 프로젝트를 동시에 관리해야 하는 대학생
- 기존 생산성 앱에서 기록·회고 입력이 귀찮아 사용을 중단한 사람
- 자신의 한계를 인식하고 계획 정확도를 점진적으로 높이고 싶은 사람

---

## 4. 핵심 가치 제안 (Core Value Proposition)
- 계획 실패를 실패로 취급하지 않고 **다음 계획 보정을 위한 데이터**로 활용
- 계획 입력과 피드백 과정에서 발생하는 **귀찮음을 구조적으로 제거**
- 기록은 최소화하고, 계획 수정에 필요한 피드백 정보는 최대화

---

## 5. MVP 기능 목록 (5개 이내)

| 번호 | 기능명 | 설명 | 포함 이유 |
|----|------|----|----|
| 1 | 내일 계획 생성 | 디폴트 계획(루틴 템플릿)이 자동 적용된 상태로 다음날 계획 생성 | 계획 입력 부담 제거 |
| 2 | 실행 후 피드백(블록식) | 미리 정의된 블록으로 빠르게 피드백 기록, 1줄 메모 선택 가능 | 기록 시간 최소화 |
| 3 | 실행 결과 요약 | 계획 대비 실제 수행 결과를 요약 표시 | 편차 인식 |
| 4 | 계획 보정 도움 | 이전 기록 기반으로 시간대/작업 순서 추천 | 계획–실행 편차 감소 |
| 5 | 기록 누적 조회 | 과거 기록을 날짜 단위로 확인 (통계 없이 요약 중심) | 자기 패턴 인식 |

---

## 6. 계획 작성 방식 (중요 설계 원칙)
- **디폴트 계획(루틴 템플릿) 자동 적용**
  - 수면, 출근/등교, 수업/근무, 식사 등
- 다음날 계획 화면 진입 시:
  - 템플릿이 자동으로 채워진 상태로 시작
  - 사용자는 추가 / 삭제 / 수정만 수행
  - 아무 조작도 하지 않으면 디폴트 계획 그대로 확정

---

## 7. 추천 기능 범위 (MVP 기준)
### 포함
- **시간대 추천**
  - 예: 운동은 밤보다 낮에 완료율이 높았음
- **작업 순서 추천**
  - 예: 공부 → 운동 순서가 더 잘 지켜졌음

### 제한 조건
- 자동 강제 배치 없음
- 추천은 항상 선택 사항
  - 버튼 1회로 적용 (예: [권장 시간대로 이동], [권장 순서로 변경])

---

## 8. 추천 로직 기준
- MVP 기본은 **룰 기반 추천**
  - 최근 7일 기록 기준
  - 완료율(준수율)을 주요 지표로 사용
- AI 사용:
  - 전면 기능 아님
  - 개인용 전제 하에 선택적으로 사용
  - 보정 문장 생성 및 설명 보조 역할

---

## 9. 피드백 방식
- 블록식 피드백 사용
  - 예상보다 오래 걸림
  - 예상보다 빨리 끝남
  - 시작이 늦어짐
  - 계획이 과했음
  - 집중 잘 됨
  - 집중이 잘 안 됨
  - 피로/컨디션 저하
  - 방해 요소 있음
  - 환경이 안 좋았음
- 텍스트 회고는 선택 사항
  - 최대 1줄 메모만 허용

※ v1.0에서는 위 블록 세트는 **고정**이며,  
사용자 추가·편집·개인화 기능은 제공하지 않는다.

---

## 10. 하지 않는 것 (Non-Goals)
- 블록 개인화 / 추가 / 편집
- 메모 기반 분석
- AI 자동 추천(강제)
- 자동 계획 생성 또는 강제 수정
- 통계 그래프, 랭킹, 비교
- 협업/공유 기능
- 고급 알림, 뽀모도로 세부 설정
- 유료화(프리미엄, 스킨, 구독 등)

---

## 11. 성공 기준 (Success Criteria)
- 피드백 입력이 사용자에게 부담스럽지 않은지 여부
- 사용자가 “이건 계속 써도 되겠다”고 느끼는지 여부
- 동일 유형의 계획에서 계획–실행 편차가 줄어드는지 여부

---

## 12. 기술 범위 (Tech Scope)
- 플랫폼: 모바일 (Android 단독 또는 Flutter 단일 코드베이스)
- 저장 방식: 로컬 저장 (SQLite / LocalStorage)
- 네트워크: 없음 (MVP에서는 오프라인)
- 외부 API: 없음 (AI는 선택적·개인용)

---

## 13. 일정 개요
- 1주차: 화면 흐름 정의 및 UI 스케치
- 2~4주차: 핵심 기능 구현
- 5주차: 실제 사용 테스트 및 불필요한 기능 제거
- 6주차: 다음 사이클(v2) 개발을 위한 정리

---

## 14. UI 설계 전 확정 전제
- 하루 단위 기준
- 시간대 슬롯: 아침 / 점심 / 오후 / 저녁 / 밤
- 추천은 자동이 아닌 제안 방식
- 계획 입력 최소, 수정 중심 UX

---

## 15. 제1목표 (명확화)
> 이 앱의 제1목표는 생산성을 극대화하는 것이 아니라,  
> **계획 → 실행 → 최소 피드백 → 다음 계획**의 사이클이  
> 실제로 지속 가능한지 검증하는 것이다.
